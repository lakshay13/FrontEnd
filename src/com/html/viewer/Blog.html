Coding Conventions:

1. Catch statement should avoid catching Throwable It is always recommended to catch Exception instead of Throwable because Throwable would catch out of Memory error, Internal Error etc and
ideally the application should not recover from these errors.
Example: We should avoid this-

✘ catch(Throwable t) {
logger.warn("Problem in the code");
}

We should be using this

✔ catch(Exception e) {
logger.warn("Problem in the code");
}
